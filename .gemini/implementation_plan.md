# Goal: Remove Hour Ticks from History Charts

Format the date columns as string labels (`YYYY-MM-DD`) prior to rendering the Daily, Weekly, and Monthly charts in the "History & Insights" dashboard to prevent Streamlit/Altair from displaying hour/time increments (like "02 AM", "04 AM") on the X-axis.

## User Review Required

> [!NOTE]
> Converting the date/period start columns to strings (`%Y-%m-%d`) tells the underlying Altair chart engine to treat the X-axis as categorical (nominal) labels rather than a linear time-scale. This forces the chart to display *only* the specific date labels, eliminating automatic sub-day time intervals.

# Goal: Persistent Login (Remember Me) Session Management

Provide a seamless auto-login experience where the user's session is remembered across browser refreshes and tab closures, bypassing the login screen and navigating directly to the tracker interface.

## User Review Required

> [!NOTE]
> We will use `streamlit-cookies-controller` to store the Supabase `access_token` and `refresh_token` securely in browser cookies.
> These cookies will have a 30-day expiration period. On every app start, the app will read these cookies and attempt to restore the session using `supabase.auth.set_session()`.

> [!IMPORTANT]
> If a user explicitly signs out, the cookies will be removed from their browser, requiring a manual login next time.

## Proposed Changes

### Dependencies

#### [MODIFY] [requirements.txt](file:///c:/Users/srpat/Projects/ByteSize-AI/requirements.txt)
- Add `streamlit-cookies-controller` to the Python requirements list.

---

### Application Logic

#### [MODIFY] [app.py](file:///c:/Users/srpat/Projects/ByteSize-AI/app.py)

- **Imports**: Add `from streamlit_cookies_controller import CookieController` and `from datetime import timedelta` (ensure both are available).
- **CSS Styling**: Inject a CSS rule inside the existing markdown `<style>` block to hide any visual elements/iframe placeholders generated by the `CookieController`:
  ```css
  div[data-testid='element-container']:has(iframe[title='streamlit_cookies_controller.cookie_controller']) {
      display: none;
  }
  ```
- **Cookie Initialization & Auto-Login Hook**:
  - Instantiate `cookie_controller = CookieController()` globally.
  - Before rendering the login tab (around line 273), check if `st.session_state.user` is not set.
  - If unset, check if `sb_access_token` and `sb_refresh_token` cookies are present.
  - If present, call `supabase_client.auth.set_session(access_token, refresh_token)`.
  - If successful, set `st.session_state.user`, `st.session_state.session`, load the user profile, update the cookies with the fresh tokens (set to expire in 30 days), and trigger `st.rerun()`.
  - If session restoration fails (e.g. expired refresh token), remove the invalid cookies.
- **Login Flow Integration**:
  - Upon successful credentials verification in `submit_login`, write the `sb_access_token` and `sb_refresh_token` to cookies (30 days expiration).
- **Signup Flow Integration**:
  - Upon successful registration (and immediate auto-login session) in `submit_signup`, write the `sb_access_token` and `sb_refresh_token` to cookies (30 days expiration).
- **Sign Out Integration**:
  - In the `sign_out()` helper function, delete `sb_access_token` and `sb_refresh_token` from browser cookies using `cookie_controller.remove()`.
